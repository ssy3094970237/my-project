# 猫咪购物园 - AI 辅助开发 Prompt 日志

本项目开发过程中，充分利用了 AI 工具辅助完成页面样式、状态管理、路由跳转及业务逻辑的编写。以下是关键功能的 Prompt 交互日志与解决文件映射。

---

## 1. 商品分类及卡片列表 UI 设计

- **解决的文件**：
  - `pages/goods/goods.wxml`
  - `pages/goods/goods.wxss`
  - `pages/goods/goods.js`
- **对应 Prompt**：
  > “我想在微信小程序中制作一个商品分类展示界面。顶部有四个分类选项卡（猫粮、罐头、零食、玩具），点击切换时能高亮并更新下方列表。每个商品卡片包括图片、标题、价格、店铺名称、服务标签（如‘极速退款’），整体风格要偏温暖、柔和的绿色。请给出 WXML、WXSS 和 JS 的初始代码。”
- **AI 原始输出简述**：
  AI 输出了基于 `flex` 布局的分类选项卡实现方案，使用数据字段 `currentTab` 存储当前分类，并配合 `data-item` 实现点击卡片时通过 `wx.navigateTo` 携带参数跳转详情页 `pages/goods/expend/expend`。

---

## 2. 购物车数量增减与总价实时计算

- **解决的文件**：
  - `pages/car/car.js`
  - `pages/car/car.wxml`
- **对应 Prompt**：
  > “在小程序的购物车页面中，如何实现选择商品、增减数量并实时刷新底部总价的功能？当购物车为空时，展示一个‘购物车空空如也’的占位提示和‘去逛逛’按钮。请提供核心 JS 逻辑及 WXML 绑定。”
- **AI 原始输出简述**：
  AI 提供了通过在 `checkboxChange`、`addNumber`、`decNumber` 事件中，遍历购物车数组计算选中商品的 `num * price` 累加值，最后调用 `this.setData({ totalPrice })` 更新视图的实现，并且用双向绑定或条件渲染 `wx:if` 控制空购物车的展示。

## 3. 收货地址簿与新增地址表单校验

- **解决的文件**：
  - `pages/address/address.js`
  - `pages/address/addAddress/addAddress.js`
- **对应 Prompt**：
  > “用户在结算前需要选择或新增收货地址。如何设计一个地址表单，要求对姓名、手机号、详细地址进行非空校验，并且手机号必须是 11 位数字？校验通过后如何将地址保存到本地缓存（`wx.setStorageSync`），并返回上一页刷新地址列表？”
- **AI 原始输出简述**：
  AI 给出了利用正则表达式 `/^1[3-9]\d{9}$/` 校验手机号的逻辑。并建议在 `onShow` 生命周期中读取 `wx.getStorageSync('addressList')` 以保证返回页面时能即时渲染最新保存的地址。

## 4. 模拟 API 数据结构及异常处理设计

- **解决的文件**：
  - `接口文档.md`
- **对应 Prompt**：
  > “我的微信小程序项目没有真正的后端接口，老师要求提供接口文档或模拟的 API，包含异常场景说明。请根据我的商城小程序，设计获取商品、提交订单、保存地址 3 个符合 RESTful 规范的 JSON 接口，并分别写出 200 成功与 400/500 异常时的 JSON 响应示例。”
- **AI 原始输出简述**：
  AI 输出了详细的 API 协议表格及对应的格式化 JSON 示例，包括参数名称、类型、必填项以及库存不足（400）、参数校验失败（422）等常见异常响应。
