# 猫咪购物园 - 前端功能实现与异常场景覆盖说明

依据实训考核细则中的 **“A. 项目功能完整度(50%) - (2) 功能实现：前端至少3个独立路由，包含UI/UX体验，健壮性（可在功能实现中增加异常场景覆盖说明）”** 要求，本篇文档对本项目的前端路由架构、UI/UX 设计细节以及异常场景的防范与处理机制进行详细说明。

---

## 一、 前端独立路由（页面）实现

本项目已实现多于 3 个独立路由，核心功能分布在以下 4 个独立路由页面中：

| 页面路径 | 路由类型 | 核心功能描述 |
| :--- | :--- | :--- |
| `pages/first/first` | TabBar 页面 (主页) | 滚动广告横幅展示、常用功能快捷导航入口、线下参观预约入口。 |
| `pages/goods/goods` | TabBar 页面 (分类商品) | 支持四大类商品（猫粮、罐头、零食、玩具）的选项卡切换及详情查看跳转。 |
| `pages/car/car` | TabBar 页面 (购物车) | 商品数量增减、勾选结算、总价实时计算，并与全局状态保持一致。 |
| `pages/address/address` | 独立路由页面 (地址管理) | 展示收货地址列表，支持选择、编辑以及新增地址。 |

---

## 二、 UI/UX 体验设计

1. **统一的色彩体系与视觉基调**：
   - 项目采用了清新柔和的天然绿灰色调（主题色为 `#93a07d`），营造出温暖、轻松的宠物主题氛围。
   - 交互文字使用深色（`#000000`），未选中/辅助文字使用灰色（`#bfbfbf`），具有极高对比度与易读性。
2. **直观的交互与动画反馈**：
   - **选项卡切换**：商品分类页采用滑块激活高亮状态，响应迅速。
   - **操作反馈**：添加购物车成功或提交订单时均伴有直观的 `wx.showToast` 弹出提示，增强操作确认感。

## 三、 健壮性与异常场景覆盖说明

为了保障前端运行的健壮性，项目针对以下多种可能出现的异常场景进行了防御性设计与友好提示：

### 1. 购物车为空时的结算异常
- **异常场景**：购物车内没有任何商品，或者用户取消勾选了所有商品，却点击了“去结算”按钮。
- **前端处理机制**：
  - 在 `pages/car/car` 页面中，如果购物车商品数量为 0，页面会自动渲染“购物车空空如也”的空状态占位图，并隐藏结算栏。
  - 当检测到结算金额为 0 或勾选商品数为 0 时，结算按钮将自动置灰，或点击时触发友好弹窗提示 `wx.showToast({ title: '您还没有选择商品哦', icon: 'none' })`，防止提交空订单到后端。

### 2. 收货地址为空时的提交订单异常
- **异常场景**：在订单支付/提交结算页中，用户未选择或未填写收货地址便点击了“确认提交”。
- **前端处理机制**：
  - 页面会显示红色的“请选择收货地址”提示框。
  - 点击“提交”时，代码会首先校验 `address` 对象的有效性，若为空则拦截提交操作，并弹出弹窗提示用户去新增或选择地址，有效阻断非法数据流出。

### 3. 地址编辑与新增的表单输入异常
- **异常场景**：用户在新增收货地址时，未填写姓名、电话或详细地址，或者填写的手机号码格式不合法（例如位数不足、包含字母等）。
- **前端处理机制**：
  - 在 `pages/address/addAddress/addAddress` 的保存事件中，利用正则表达式 `/^1[3-9]\d{9}$/` 对手机号进行强校验。
  - 任何一项校验失败，均会使用 `wx.showToast` 弹出精准的错误提示（如“请输入正确的手机号”），并且不予保存，确保了数据的健壮性。

### 4. 本地缓存读取失败/溢出异常
- **异常场景**：读取或写入用户订单数据、收货地址列表时，系统磁盘已满或发生内部读取故障。
- **前端处理机制**：
  - 核心读写逻辑均采用 `try...catch` 语句包裹：
    ```javascript
    try {
      var addressList = wx.getStorageSync('addressList') || [];
      // 业务逻辑...
    } catch (e) {
      console.error("读取地址缓存失败:", e);
      wx.showToast({ title: '数据读取失败，请稍后重试', icon: 'none' });
    }
    ```
  - 这保证了即使本地缓存失效，小程序主页面也绝不会产生白屏或崩溃。
